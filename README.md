# Routsol Web — Персональный гид по отдыху

Веб-приложение, которое подбирает места и домашние активности на основе психологического профиля пользователя, его текущего состояния, местоположения, времени суток и погоды. Включает социальную сеть: систему друзей, приглашения на активности, историю взаимодействий и настройки приватности.

**Демо:** http://routsol.silaeder.space

---

## Быстрый старт

### Docker (рекомендуется)

```bash
# 1. Клонировать репозиторий
git clone <repository-url>
cd RoutsolWeb

# 2. Настроить переменные окружения
copy .env.example .env
# Отредактируйте .env при необходимости

# 3. Запустить все сервисы
docker-compose up -d

# 4. Открыть в браузере
# http://localhost:5000
```

## Основные возможности

### Для пользователей
- **Психологическая анкета** — 21 вопрос для определения профиля
- **Умные рекомендации** — подбор мест на основе состояния и местоположения
- **Домашние активности** — 30+ вариантов для отдыха дома
- **Социальная сеть** — друзья, приглашения, совместные активности
- **Темная тема** — автоматическое переключение
- **Адаптивный дизайн** — работает на всех устройствах

### Технические особенности
- **Безопасность** — защита от SQL Injection, XSS, CSRF
- **Производительность** — индексы БД, оптимизированные запросы
- **Docker** — запуск одной командой
- **PostgreSQL** — надежное хранение данных
- **Современный UI** — CSS переменные, плавные анимации

## Описание файлов

### `app.py` — главный файл приложения

Точка входа. Содержит инициализацию Flask, конфигурацию базы данных, все HTTP-маршруты и вспомогательные функции.

**Инициализация:**
- Создаёт экземпляр Flask и подключает SQLAlchemy
- При первом запуске создаёт все таблицы (`db.create_all()`) и заполняет вопросы анкеты (`init_questions()`)
- Загружает переменные окружения через `python-dotenv`

**Вспомогательные функции:**
- `login_required` — декоратор, перенаправляющий неавторизованных пользователей на страницу входа
- `get_current_user()` — возвращает объект текущего пользователя из сессии
- `inject_user()` — контекстный процессор, делающий `current_user` и `yandex_maps_key` доступными во всех шаблонах
- `calculate_profile_scores(answers_dict)` — вычисляет баллы по 6 психологическим шкалам из словаря ответов
- `calculate_restrictions(answers_dict)` — определяет флаги ограничений (алкоголь, бюджет, физические) из ответов на вопросы 19–21
- `get_time_of_day()` — определяет время суток (morning/afternoon/evening/night) по текущему часу
- `fetch_weather(lat, lon)` — запрашивает текущую погоду через Open-Meteo API (бесплатно, без ключа)
- `find_nearby_places(lat, lon, state_id)` — ищет места рядом через Nominatim API и сохраняет в БД
- `infer_place_attributes(place_type, api_data)` — определяет психологические атрибуты места по его типу

**Маршруты авторизации:**
- `GET/POST /register` — регистрация нового пользователя
- `GET/POST /login` — вход в аккаунт
- `GET /logout` — выход

**Основные маршруты:**
- `GET /` — главная страница
- `GET /questionnaire` — анкета
- `POST /api/save_answer` — AJAX-сохранение ответа на вопрос
- `POST /submit_questionnaire` — завершение анкеты, подсчёт профиля и ограничений
- `GET /profile` — собственный психологический профиль
- `GET /ask_state` — форма текущего состояния
- `POST /save_state` — сохранение состояния, получение погоды, поиск мест
- `GET /recommendations` — страница рекомендаций
- `GET /place/<id>` — детали места
- `GET /home-activity/<id>` — детали домашней активности
- `GET /history` — история рекомендаций
- `GET/POST /settings` — настройки и приватность

**Социальные маршруты:**
- `GET /social` — главная социального раздела
- `GET /social/users` — поиск пользователей
- `GET /social/user/<id>` — публичный профиль пользователя
- `POST /social/friend_request/<id>` — отправить запрос в друзья
- `POST /social/friend_request/<id>/respond` — принять/отклонить запрос
- `POST /social/invite` — отправить приглашение на активность
- `POST /social/invite/<id>/respond` — принять/отклонить приглашение
- `GET /api/social/notifications` — количество непрочитанных уведомлений (AJAX)
- `GET /api/friends` — список друзей в JSON (для модального окна шаринга)

### `models.py` — модели базы данных

Использует SQLAlchemy ORM. Каждый класс соответствует таблице в SQLite.

**`User`** — пользователь системы
- `id`, `username`, `email`, `password_hash` — базовые поля авторизации
- `age`, `gender` — демографические данные (заполняются в анкете)
- `set_password()` / `check_password()` — хэширование через `werkzeug.security`
- `get_latest_state()` — последнее состояние пользователя
- `get_friends()` — список принятых друзей (объединяет входящие и исходящие принятые запросы)
- `is_friend_with(uid)` — проверка дружбы
- `friend_request_status(uid)` — объект запроса в друзья между двумя пользователями

**`UserState`** — снимок состояния пользователя в момент запроса рекомендации
- `fatigue_level` / `fatigue_value` — уровень усталости (high/medium/low и числовое значение)
- `latitude`, `longitude`, `address` — местоположение
- `mood`, `time_of_day`, `with_company` — контекст
- `weather_temp`, `weather_code`, `weather_desc`, `weather_icon` — данные погоды от Open-Meteo

**`PsychologicalProfile`** — результаты психологической анкеты
- 6 шкал (3–15 баллов): `extraversion`, `openness`, `sensation_seeking`, `conscientiousness`, `proactivity`, `neuroticism`
- 3 флага ограничений: `no_alcohol`, `low_budget`, `physical_limits`

**`Question`** — вопросы анкеты (21 штука, хранятся в БД)
- `scale` — к какой шкале относится вопрос
- `options` — JSON с вариантами ответов (1–5)

**`UserAnswer`** — сырые ответы пользователя на анкету
- `answers_data` — JSON `{question_id: answer_value}`
- `completed` — завершена ли анкета

**`Place`** — найденное место для отдыха
- Координаты, тип, адрес, расстояние
- Психологические атрибуты: `atmosphere`, `activity_level`, `avg_price`, `capacity`
- `match_score` — оценка соответствия профилю (0–100)

**`FriendRequest`** — запрос в друзья
- `sender_id`, `receiver_id`, `status` (pending/accepted/declined)

**`ActivityInvite`** — приглашение на активность
- `activity_type` (place/home_activity/custom), `activity_name`, `activity_icon`
- `status` (pending/accepted/declined)

**`Meeting`**, **`MeetingParticipant`** — модели встреч (таблицы сохранены в БД, функционал упрощён до редиректа)

**`UserSettings`** — настройки приватности пользователя
- `show_age`, `show_email`, `show_gender` — видимость личных данных
- `show_profile_scores` — видимость результатов анкеты
- `show_restrictions` — видимость ограничений
- `allow_friend_requests` — разрешить запросы в друзья

### `ai_ranking.py` — алгоритм ранжирования

Эвристический rule-based алгоритм. Не использует ML — все правила написаны вручную.

**`_weather_category(code)`** — переводит WMO-код погоды в категорию (clear/cloudy/rain/snow/storm/fog)

**`calculate_place_score(profile, state, place, user_age)`** — вычисляет оценку места (0–100):
- Усталость: при высокой — тихие места +20, шумные −20
- Экстраверсия: высокая — большие заведения +10, низкая — тихие места +10
- Поиск ощущений: высокий — активные места +15
- Невротизм: высокий — тихие +15, шумные −15
- Открытость: высокая — музеи и кино +10
- Компания: один — тихие +5, с друзьями/семьёй — большие +10
- Настроение: грусть → кафе/парки, злость → спортзал, тревога → тихие, скука → кино/музей
- Расстояние: <500м +10, <1км +5, >2км −5
- Погода: дождь/снег → закрытые места +15, ясно → парки +12
- Время суток: ночь → парки −15, утро → кафе +8, вечер → бары/кино +8
- Возраст: <18 → парки/кино/кафе +8, ≥60 → тихие/пассивные +10
- Бюджет: дешёвые места +15 при `low_budget`

**`rank_places(profile, state, places, user_age)`** — фильтрует и сортирует места:
- Полностью исключает бары для <18 лет и при `no_alcohol`
- Исключает спортзалы при `physical_limits`
- Исключает дорогие места при `low_budget`

**`get_recommendation_explanation(profile, state, place)`** — генерирует текстовое объяснение рекомендации

**`get_home_activities(profile, state)`** — подбирает топ-3 домашних активности из 30 кандидатов:
- Фильтрует по компании (alone/partner/friends/family/any)
- Исключает физические активности при `physical_limits`
- Оценивает по усталости (+25), настроению (+15), совпадению компании (+10), психологическому профилю (+10 за каждую черту), плохой погоде (+12)

### `api_integrations.py` — интеграции с API поиска мест

**`PlaceSearchAPI`** — базовый класс с методом `calculate_distance()` (формула Haversine)

**`NominatimAPI`** — поиск через OpenStreetMap Nominatim (бесплатно, без ключа, лимит 1 запрос/сек)

**`TwoGISAPI`** — поиск через 2GIS (требует `TWOGIS_API_KEY`)

**`YandexMapsAPI`** — поиск через Яндекс Карты (требует `YANDEX_API_KEY`)

**`get_place_search_api(api_type)`** — фабричная функция, возвращает нужный API по строке ('nominatim'/'2gis'/'yandex')

## Шаблоны (templates/)

### `_macros.html` — переиспользуемые Jinja2-макросы (иконки)

Макросы подключаются в шаблонах через `{% from '_macros.html' import ... %}`.

### `base.html` — базовый шаблон

Все остальные шаблоны наследуют его через `{% extends "base.html" %}`.

Содержит:
- Подключение CSS (`style.css`, `place-icons.css`), Font Awesome, Google Fonts (Inter)
- Навигационную шапку с адаптивным меню
- Группу пользователя: имя (обрезается до 14 символов), кнопка настроек, кнопка выхода
- Переключатель темы (плавающая кнопка в правом нижнем углу, сохраняет выбор в `localStorage`)
- Flash-сообщения с категориями (success/error/info)
- JavaScript: инициализация темы до рендера страницы (в `<head>`), опрос уведомлений каждые 30 секунд

### `index.html` — главная страница

Показывает разные карточки-действия в зависимости от состояния пользователя:
- Гость → "Создать аккаунт" + "Войти"
- Авторизован без анкеты → "Заполнить анкету" (главная карточка)
- Авторизован с анкетой → "Получить рекомендацию", "Социальное", "Профиль", "История"
- Если есть последнее состояние → карточка "Последние рекомендации" с датой и настроением

Блок "Как это работает" показывается только тем, кто ещё не заполнил анкету.

### `questionnaire.html` — психологическая анкета

21 вопрос, разбитых на 7 шкал:
- Экстраверсия (3 вопроса)
- Открытость опыту (3 вопроса)
- Поиск ощущений (3 вопроса)
- Сознательность (3 вопроса)
- Проактивность (2 вопроса)
- Невротизм / потребность в покое (3 вопроса)
- Ограничения (3 вопроса: алкоголь, физические, бюджет)

Процесс: сначала демографический блок (возраст + пол), затем вопросы по одному с шкалой Ликерта 1–5. Каждый ответ сохраняется через AJAX (`/api/save_answer`) без перезагрузки страницы. Прогресс-бар показывает текущий вопрос.

### `ask_state.html` — форма текущего состояния

Пользователь указывает:
1. **Местоположение** — вручную или через `navigator.geolocation`. 
2. **Уровень усталости** — три карточки (полон энергии / умеренно устал / очень устал)
3. **Настроение** — 11 вариантов
4. **Компания** — 5 вариантов 

При отправке: сохраняется `UserState`, запрашивается погода через Open-Meteo, ищутся места через Nominatim.

### `recommendations.html` — страница рекомендаций

**Логика отображения:**
- Если `fatigue_level == 'high'` → режим "остаться дома": показывается баннер с иконкой погоды и объяснением, карта и места скрыты
- Иначе → карта + топ-5 мест

**Шапка состояния** показывает: погоду (иконка + описание + температура), время суток, усталость, настроение, компанию.

**Карта:** Яндекс Карты если задан `YANDEX_MAPS_KEY`, иначе Leaflet/OpenStreetMap.

**Карточки мест:** иконка типа (Font Awesome), название, тип, соответствие профилю, расстояние, объяснение рекомендации, атрибуты (атмосфера, активность, цена, вместимость), адрес, кнопка "Позвать друга".

**Домашние активности:** топ-3 карточки с иконкой, названием, описанием, типом компании, процентом совпадения, кнопкой "Поделиться".

**Модальное окно шаринга:** выбор друга из списка + сообщение → отправляет `ActivityInvite`.

## База данных

Приложение использует **PostgreSQL** для хранения данных.

### Настройка PostgreSQL

1. Установите PostgreSQL: https://www.postgresql.org/download/
2. Создайте базу данных:
   ```bash
   psql -U postgres -c "CREATE DATABASE routsol;"
   ```
3. Настройте подключение в `.env`:
   ```env
   DATABASE_URL=postgresql://postgres:ваш_пароль@localhost/routsol
   ```

## Алгоритм подбора домашних активностей

30 активностей разбиты на 5 категорий по типу компании:
- **Одиночные (12):** медитация, книга, дневник, ванна, рисование, онлайн-курс, тренировка, йога, подкаст, фото, уборка, прогулка по квартире
- **С партнёром (4):** фильм вдвоём, готовка вместе, настолка на двоих, домашний спа
- **С друзьями (5):** онлайн-игры, видеозвонок, watch-party, викторина, совместная готовка
- **С семьёй (4):** семейный киновечер, готовка с семьёй, настолки, творчество
- **Любая компания (3):** музыка, готовка, видеоигры

Алгоритм фильтрует по компании, затем оценивает каждую активность и возвращает топ-3.

## 🔧 Команды разработчика

### Локальная разработка
```bash
# Запуск в режиме разработки
python app.py

# Применение индексов
python apply_indexes.py

# Создание нового пользователя
python create_user.py
```

### Docker
```bash
# Сборка и запуск
docker-compose up --build

# Просмотр логов
docker-compose logs -f web

# Выполнение команд в контейнере
docker-compose exec web python generate_test_data.py

# Остановка
docker-compose down
```

### База данных
```bash
# Подключение к PostgreSQL
docker-compose exec db psql -U postgres -d routsol

# Бэкап
docker-compose exec db pg_dump -U postgres routsol > backup.sql

# Восстановление
docker-compose exec -T db psql -U postgres routsol < backup.sql
```
