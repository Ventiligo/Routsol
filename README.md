# Routsol Web — Персональный гид по отдыху

Веб-приложение для подбора мест и домашних активностей на основе психологического профиля пользователя, его состояния, местоположения и погоды. Включает социальную сеть с системой друзей и приглашений.

---

## Быстрый старт

### Docker (рекомендуется)

```bash
# 1. Клонировать репозиторий
git clone <repository-url>
cd Routsol

# 2. Настроить переменные окружения
copy .env.example .env
# Отредактируйте .env при необходимости

# 3. Запустить все сервисы
docker-compose up -d

# 4. Открыть в браузере
# http://localhost:5000
```

### Локальная разработка

```bash
# Установить зависимости
pip install -r requirements.txt

# Настроить PostgreSQL
createdb routsol

# Запустить приложение
python app.py
```

## База данных

Приложение использует **PostgreSQL** для хранения данных.

### Настройка PostgreSQL

1. Установите PostgreSQL: https://www.postgresql.org/download/
2. Создайте базу данных:
   ```bash
   psql -U postgres -c "CREATE DATABASE routsol;"
   ```
3. Настройте подключение в `.env`:
   ```env
   DATABASE_URL=postgresql://postgres:ваш_пароль@localhost/routsol
   ```

### Основные таблицы

| Таблица | Описание |
|---|---|
| `users` | Пользователи: авторизация, демография |
| `psychological_profiles` | Результаты анкеты: 6 шкал + 3 ограничения |
| `user_answers` | Сырые ответы на вопросы анкеты (JSON) |
| `questions` | 21 вопрос анкеты |
| `user_states` | Состояния: усталость, настроение, координаты, погода |
| `places` | Найденные места с атрибутами и оценкой |
| `friend_requests` | Запросы в друзья |
| `activity_invites` | Приглашения на активности |
| `user_settings` | Настройки приватности |

## Алгоритм рекомендаций

### Психологические шкалы (21 вопрос)
- **Экстраверсия** (3 вопроса) — предпочтение общения и шумных мест
- **Открытость опыту** (3 вопроса) — готовность к новому
- **Поиск ощущений** (3 вопроса) — потребность в адреналине
- **Сознательность** (3 вопроса) — планирование и организованность
- **Проактивность** (2 вопроса) — инициативность
- **Невротизм** (3 вопроса) — потребность в покое и тишине
- **Ограничения** (3 вопроса) — алкоголь, физические, бюджет

### Факторы ранжирования мест
- Психологический профиль пользователя
- Текущее состояние (усталость, настроение)
- Погодные условия
- Время суток
- Тип компании (один, с друзьями, семья)
- Расстояние от пользователя
- Возрастные ограничения

### Домашние активности
30 активностей по категориям компании:
- **Одиночные** (12): медитация, чтение, рисование, йога
- **С партнёром** (4): фильм вдвоём, готовка, настолки
- **С друзьями** (5): онлайн-игры, видеозвонок, викторина
- **С семьёй** (4): семейный киновечер, творчество
- **Любая компания** (3): музыка, готовка, видеоигры

## Технологический стек

### Backend
- **Flask 3.0** — веб-фреймворк
- **SQLAlchemy 2.0** — ORM
- **PostgreSQL 15** — база данных
- **Flask-WTF** — защита от CSRF
- **Flask-Limiter** — rate limiting
- **Werkzeug** — хеширование паролей

### Frontend
- **Jinja2** — шаблонизатор
- **CSS3** — стили с переменными
- **JavaScript (Vanilla)** — интерактивность
- **Chart.js** — визуализация профиля
- **Font Awesome 6** — иконки

### Внешние API
- **Nominatim (OpenStreetMap)** — поиск мест (бесплатно)
- **Open-Meteo** — погода (бесплатно)
- **Яндекс Карты JS API** — интерактивная карта

### DevOps
- **Docker** — контейнеризация
- **Docker Compose** — оркестрация
- **Gunicorn** — WSGI сервер (production)

## Безопасность

- **Защита от SQL Injection** — использование SQLAlchemy ORM
- **Защита от XSS** — экранирование в Jinja2 шаблонах
- **Защита от CSRF** — Flask-WTF токены
- **Rate Limiting** — ограничение запросов на авторизацию
- **Безопасное хеширование паролей** — Werkzeug + соль
- **Валидация входных данных** — проверка на сервере
- **Безопасные заголовки HTTP** — X-Content-Type-Options, X-Frame-Options

## Основные страницы

- **Главная** (`/`) — карточки действий в зависимости от статуса пользователя
- **Анкета** (`/questionnaire`) — 21 вопрос с AJAX-сохранением
- **Состояние** (`/ask_state`) — форма усталости, настроения, местоположения
- **Рекомендации** (`/recommendations`) — места и домашние активности
- **Профиль** (`/profile`) — результаты анкеты с диаграммой
- **Социальное** (`/social`) — друзья, запросы, приглашения
- **История** (`/history`) — прошлые рекомендации
- **Настройки** (`/settings`) — приватность и настройки аккаунта

## Известные ограничения

1. **Алгоритм рекомендаций** — rule-based, не использует ML
2. **Поиск мест** — ограничен возможностями Nominatim API
3. **Геолокация** — требует разрешения браузера
4. **Карты** — требуется API ключ Яндекса для полной функциональности
